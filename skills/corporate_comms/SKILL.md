# Corporate Comms Skill

Drafts professional communications for Arnold's engineering management role at Bally's International.

## When to Use

- Arnold asks to draft an email, message, or status update for Bally's
- Arnold needs help preparing talking points for a Bally's meeting
- Arnold asks for help with team communications (announcements, feedback, reviews)
- Before or during the 1PM–8PM Bally's work block

## Context

- Read `context/ballys.md` for role details, team structure, and key contacts
- Arnold cannot paste AI-generated content directly into Bally's tools in a way that exposes AI usage — drafts should sound naturally like Arnold
- All output is for Arnold to review and manually transfer to Outlook/Teams

## Communication Types

### 1. Email Drafts
- **Tone:** Professional, clear, warm but not overly casual
- **Structure:** Lead with the key point/ask, then context, then next steps
- **Length:** Match the situation — shorter is better unless complexity demands more
- **Sign-off:** Arnold's preference (to be learned over time)

### 2. Status Updates
- **Format:** Bullet points with clear headers per team
- **Content:** What was done, what's in progress, blockers, next steps
- **Audience-aware:** Adjust detail level for leadership vs. peers vs. direct reports

### 3. Meeting Talking Points
- **Format:** Ordered list of discussion topics with key points under each
- **Include:** Questions to ask, decisions to make, information to share

### 4. Team Messages (Teams/Slack-style)
- **Tone:** Slightly more casual than email, but still professional
- **Length:** Short and direct — optimized for chat context

### 5. Feedback & Reviews
- **Structure:** Situation → Behavior → Impact framework
- **Tone:** Constructive, specific, and respectful
- **Include both:** What's working well + areas for growth

## Output Format

```
## [Type]: [Subject/Purpose]
**To:** [Recipient(s)]
**Context:** [One line on why this is being sent]

---

[Draft content]

---

**Notes for Arnold:** [Any suggestions, alternatives, or things to double-check]
```

## Notes

- Always ask Arnold for context before drafting — don't assume details about Bally's internal matters
- Never include proprietary Bally's information in memory or context files
- Drafts are disposable — they exist only for Arnold to use in the moment
- If Arnold develops patterns (e.g., weekly status format), capture the template in `resources/templates/`
