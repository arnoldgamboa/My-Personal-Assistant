# Sermon Prep Workflow

Run this workflow when Arnold is preaching on Sunday. Starts Thursday, runs intermittently through Sunday.

## Pre-requisite

- Arnold has confirmed he is preaching this Sunday
- Arnold has identified the passage or topic (or needs help choosing)
- `context/lifecity_church.md` has been read for methodology and series context

## Steps

### Thursday — Research & Foundation

1. **Confirm the Text**
   - Ask Arnold: "What passage or topic are you preaching on Sunday?"
   - If part of a series, check `context/lifecity_church.md` for the series plan

2. **Run Exegesis**
   - Invoke `agents/sermon_agent.md` for passage research
   - Study context: historical setting, literary structure, theological themes
   - Identify cross-references and key connections
   - Surface original language insights where helpful

3. **Identify the Big Idea**
   - Distill the passage into one core sentence
   - Present to Arnold: "Here's the big idea I'm seeing — does this resonate?"

4. **Initial Outline**
   - Draft a rough outline with 2–4 main points
   - Include: introduction hook, main points, transitions, application areas
   - Present for Arnold's feedback

### Friday — Development

5. **Refine the Outline**
   - Incorporate Arnold's Thursday feedback
   - Develop each point more fully
   - Add supporting scripture and connections

6. **Source Illustrations**
   - Find stories, analogies, and real-world examples for each point
   - Prioritize: culturally relevant, fresh (not clichéd), emotionally resonant
   - Present 2–3 options per point for Arnold to choose from

7. **Develop Applications**
   - Craft practical, specific applications
   - Balance: individual + communal, internal + external
   - Connect ancient text to the congregation's real life

### Saturday — Finalization

8. **Final Review**
   - Produce the complete sermon prep document
   - Check: Does every section serve the big idea?
   - Check: Is the flow logical and natural?
   - Check: Are transitions smooth?

9. **Save the Final Draft**
   - Save to `resources/drafts/sermon-YYYY-MM-DD-[short-title].md`
   - Log in `memory/` that sermon prep is complete

### Sunday — Delivery Support

10. **Quick Reference (if requested)**
    - Provide a condensed bullet-point version of key points
    - Last-minute adjustments only — no major restructuring

## Completion

Report: "Sermon prep complete for Sunday. Draft saved to `resources/drafts/`."

## Skip Conditions

- If Arnold says he is **not preaching** this Sunday, skip this entire workflow
- If Arnold is preaching but wants minimal help, scale back to just the outline phase
